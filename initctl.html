Well, this gets really monotonous over the time ! Creating the same initctl scripts again and again

What are these initctl scripts anyway?

In short, a service script that runs at linux startup. These scripts are /etc/init/*.conf files


You invoke a script with:
initctl start service1

How do you create one?

Let's say you want to run a node.js app on system startup you would do. 
echo "exec node /home/scriptmaster/service1.js" > /etc/init/service1.conf

And to list all service scripts, simply issue the command:


initctl list

To check the status of ur service, do: 

initctl status service1

to stop the service, do: 

initctl stop service1

For automatic startup add a line at the top in the service1.conf file:
start on started rc RUNLEVEL=[2345]
